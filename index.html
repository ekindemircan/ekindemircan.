Hi, I'm Ekin!

I started my journey in architecture, where I learned how people move through spaces and how design shapes experience. Along the way, I discovered a love for games — how they engage, challenge, and connect us. That curiosity eventually led me to UX design, where I found the perfect blend of my interests.

Now, I focus on designing user experiences for mobile games. I enjoy creating interfaces that feel intuitive, playful, and meaningful. My background helps me see design from both a creative and structured lens — and I love turning complex ideas into simple, fun interactions.

Find Hidden Objects

As a UI/UX designer on this project, I work across the full spectrum of the game's interface — from core navigation to in-game popups. Below is a selection of screens from one game, showcasing how I approach visual consistency, player experience, and monetization UX within a single product. Each screen is part of a cohesive design system built to feel familiar, fast, and fun.

01 — Home Screen
Main Gameplay Hub
The central hub showing level progress, daily boosters, and coin balance. Events are surfaced as a dedicated button directly on the home screen for quick access — replacing buried menu navigation with a single, visible entry point.
Core Loop Navigation Events Booster UI
02 — Shop
In-Game Store
Each package uses a distinct color to draw the eye individually, while packages are grouped logically within the layout. This visual hierarchy makes it easy to browse, compare, and act — without overwhelming the player.
Color Grouping IAP Visual Hierarchy Monetization
03 — Achievements
Progress & Badge System
A scrollable list layout that prioritises readability over decoration. Each achievement is easy to scan at a glance — progress bars, reward indicators, and badge tiers are all clearly visible. Claimable rewards use a high-contrast green CTA to stand out immediately.
List Layout Readability Gamification Retention
04 — Settings
Preferences & Accessibility
A monochrome, familiar interface format — no visual noise. Users can quickly scan and find what they need. Color Blind Assist is included as a first-class toggle, and the grouped sections keep related actions together without clutter.
Accessibility Monochrome UI Scanability Toggle Design
Popup System

These four screens share the same base popup design language — consistent components, layout structure, and visual tone. Each adapts slightly for its function, but the underlying system ensures players always feel at home, no matter which popup they encounter.

End State
Event Fail Screen
Shown at the end of a failed event run. Displays at-risk milestone rewards on a progress track with a rewarded ad recovery CTA and a restart fallback.
Event
Event Start Popup
Shown when a player enters an active event. Features a countdown timer, event icon, and a brief description — giving context before the action begins.
Offer
No Ads Offer
A conversion-focused IAP popup with a bold visual, clear value proposition, and a single green CTA. Minimal friction, maximum clarity.
Profile
Edit Profile Popup
Username and avatar are editable from a single popup — both actions in one place, reducing steps. Clear active state in the avatar grid.
